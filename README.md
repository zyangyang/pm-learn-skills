# PM Learn - 产品经理技术学习助手

> 帮产品经理深度学习技术文档的 TRAE 斜杠命令（Slash Command）。
> 通过「提问 → 讲解 → 总结 → 归档」的四步闭环，把复杂技术概念转化为可决策、可复用的结构化知识。

## 这是什么

`pm-learn` 是一个面向产品经理的 TRAE IDE 技能，安装后可通过 `/pm-learn` 调用。它不是普通聊天机器人，而是一个**结构化学习框架**：

- 拿到技术文档/链接/概念 → 按五层结构讲解
- 回答"和 XX 有什么区别" → 多维度对比表
- 回答"成本/性能" → 量化数据 + 优化策略
- 回答"对产品设计的影响" → 交互范式变化 + 场景对比 + UX 模式 + 风险应对
- 全程自动归档为 Markdown 总结

## 安装

将本仓库的 `SKILL.md` 放到 TRAE 的 skills 目录下：

| 平台 | 路径 |
| --- | --- |
| Windows | `%USERPROFILE%\.trae-cn\skills\pm-learn\SKILL.md` |

或者在 TRAE IDE 的命令面板中执行"导入技能"并选择本仓库。

## 使用

在 TRAE 对话框中直接输入：

```
/pm-learn https://docs.langchain.com/oss/python/langchain/mcp 给我讲解一下
```

或粘贴任意技术主题/文档内容后追加"按 pm-learn 框架讲解"。

## 五层讲解框架

| 层级 | 关注点 | 典型提问 |
| --- | --- | --- |
| 1. 概念层 | What | 这是什么？解决什么问题？和类似概念区别？ |
| 2. 原理层 | Why | 为什么需要？核心机制？代价/风险？ |
| 3. 应用层 | How | 应用场景（≥3）？何时用/不用？成本收益？ |
| 4. 实战层 | Do | 最小示例？常见陷阱？系统集成？ |
| 5. UX 影响层 | Impact | 交互范式变化？场景对比？可复用 UX 模式？风险应对？ |

## 归档规范

每次学习结束会自动生成 `[主题]-讲解总结.md`，统一九段式结构：

1. 什么是 [主题]？
2. [主题] vs [相关概念] 对比
3. 产品经理场景举例
4. [核心机制/原理] 详解
5. 产品经理决策框架
6. [主题] 核心功能一览
7. 实战：如何开发/使用
8. 对产品设计/UX 的影响
9. 关键总结
